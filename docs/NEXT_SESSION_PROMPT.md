### تقنيات مهمة (من الذاكرة)
- **CI تلقائي مفعّل:** أي push يغيّر `supabase/functions/**` على main → ينشر الـEdge Functions تلقائياً (workflow + secrets مضبوطة). فقط ادفع.
- **تشغيل SQL:** عبر `POST https://api.supabase.com/v1/projects/fghdumrgimoeqsafdhhh/database/query` بتوكن `SUPABASE_ACCESS_TOKEN` (sbp_…). الأحرف العربية في curl تتشوّه → اكتب JSON لملف بترميز UTF-8 ثم `--data-binary @file`.
- **قاعدة حرجة:** جدول `profiles` يستخدم column-level grants → أي عمود جديد يحتاج `GRANT SELECT (col) ON profiles TO anon, authenticated;` وإلا يكسر تسجيل الدخول.
- **الموظفون على PIN session** (auth.uid()=null) → أي جدول يكتب له موظف يحتاج RLS متساهل + GRANT.
- **اختبار حيّ بلا PIN:** أضف مؤقتاً بآخر `authStore.js`: `if (import.meta.env.DEV) window.__authStore = useAuthStore;` ثم احقن جلسة عبر `setSession({...})`+`setReady(true)` وانتقل للمسار، **واحذف السطر بعد الاختبار**. (config المعاينة: `.claude/launch.json` باسم `lowes-dev`، port 5179.)
- نظام الصلاحيات: `src/data/permissions.js` (`PERMISSIONS` + `ROLE_PERMISSIONS` + `profiles.extra_permissions`/`denied_permissions`) + hook `usePermissions()`.

## المهام المطلوبة

### 1) 🔴 إصلاح عاجل — تسجيل الدوام للورديات الليلية
**المشكلة (شكا منها موظفون كثر):** موظف تنتهي ورديته الساعة **1:00 (بعد منتصف الليل)** — لا يستطيع تسجيل **دخول في نفس اليوم الساعة 10:00 صباحاً**. النظام يرفض لأن منطق الحضور يربط بالـ«يوم» وبصفّ `out` سابق.
- **افحص:** `src/screens/AttendanceScreen.jsx` (منطق `alreadyOut`, `MIN_MINUTES_BEFORE_CHECKOUT`, عمود `day`/`date`, و`UNIQUE(employee_name, date)` على جدول `attendance`).
- الورديات العابرة لمنتصف الليل: خروج 1:00 ص يخصّ وردية «أمس»، فيجب ألّا يمنع دخول جديد اليوم 10:00 ص.
- **الحل المتوقع:** فصل «وردية» عن «تاريخ تقويمي» (أو السماح بسجل دخول جديد طالما الخروج الأخير يخصّ وردية مغلقة سابقة). استخدم `systematic-debugging`: استنسخ الحالة فعلياً (احقن سجل out بـ1:00 ثم جرّب دخول 10:00) قبل الإصلاح، واختبر بعده.

### 2) 🏗️ إعادة تصميم نظام الأدوار والصلاحيات (احترافي)
موجود لكن غير احترافي — أعِد النظر فيه.
- **الأدمن = حسام، أماني، ريم** فقط: يحدّدون لكل شخص **دوره ومسؤوليته** + **ما يقدر يعمله** + الميزات/الصلاحيات التي يستخدمها.
- واجهة إدارة صلاحيات واضحة واحترافية (في `AdminUsersScreen` أو شاشة مخصّصة): مجموعات صلاحيات منطقية، وصف لكل صلاحية بالعربي، قوالب أدوار جاهزة، معاينة «ماذا يرى هذا المستخدم».
- ابدأ بـ`brainstorming` لتصميم البنية، ثم نفّذ. اعتمد على `permissions.js` الموجود ووسّعه بدل إعادة البناء من الصفر.

### 3) 📖 دليل استخدام داخل التطبيق — per-role (ديناميكي)
الهدف: **بدون شرح فردي لكل موظف**. كل مستخدم — حسب دوره وصلاحياته — يفتح له شرحاً عن: **ما المطلوب منه، كيف يستخدم الميزة، ولماذا**. للموظف وللإدارة.
- ديناميكي: يقرأ صلاحيات المستخدم (من المهمة #2) ويعرض فقط ما يخصّه.
- محتوى عملي لكل ميزة (الطلبات، الحضور، الحملات، المحاسبة، المهام، التدريب...). أيقونة «❓ دليل الاستخدام» + شرح مضمّن (tooltip/مودال) بكل شاشة.
- **نفّذها بعد #2 و#4 و#5** ليغطّي الدليل الحالة النهائية.

### 4) 📣 منظومة متابعة الحملات (للميديا باير OLGA + الموظفين)
- **الميديا باير (OLGA)** تنشئ حملة داخل التطبيق + **عدّاد إعلانات** داخل الحملة.
- **الموظفون** يسجّلون لكل إعلان يومياً: **كم رسالة وصلت**، **كم شخص اشترى**، + **ملاحظة عن الإعلان**.
- **الحملة الواحدة قد يشترك فيها 3 أشخاص** — كل واحد يسجّل **حسب ورديته**.
- **OLGA ترى:** هل الحملة يسجّلها المكلّفون بها فعلاً (متابعة الالتزام) + أداء كل إعلان.
- **تكلفة الحملة:** **الموظفون لا يرونها**؛ **الميديا باير + الأدمن يرونها** (صلاحية `VIEW_CAMPAIGN_COST` مثلاً).
- ابنِ على `CampaignsScreen` الموجودة. `brainstorming` أولاً للبنية (جداول campaigns/campaign_ads/ad_daily_logs)، ثم نفّذ مع ربط الصلاحيات من #2.

### 5) 💰 نظام محاسبة احترافي
- ارفع المستوى لنظام محاسبة **شركة محترفة** (راجع memory `treasury_voucher` + `accounting_lozy_syria` — يوجد أساس محافظ خزينة ووصل دفع).
- **أنواع المحافظ:** **كاش (Cash)** · **شام كاش (Sham Cash)** · **بنك (Bank)**.
- **العملات:** **دولار (USD)** و**ليرة (SYP/TRY)**.
- تقارير، حركات، أرصدة لكل محفظة×عملة، سندات قبض/صرف، ربط بالطلبات/العمولات. `brainstorming` لتحديد النطاق «الاحترافي» مع المالك المنطقي ثم بناء تدريجي مختبَر.

### 6) 🤖 تطوير لوزي كوكيل مساعد متقدّم
- لوزي (`ai-assistant` edge fn، Claude tool-use موجود) → **وكيل مساعد حقيقي**: ينفّذ، **يخطّط، يجدول، ينشئ مهام كثيرة**، مساعد للموظفين.
- وسّع الأدوات (planning/scheduling/bulk task creation/تذكيرات) مع **بوابة الصلاحيات** (least-privilege، راجع `lozy_agent_smart_quiz`). اربطه بالأدوار من #2.
