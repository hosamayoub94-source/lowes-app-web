# منظومة المخازن — دقّة الجرد + توقيت الخصم + بنية المخازن (يونيو 2026)

> طلب المالك (14 يونيو 2026): الجرد في «مبيعات سوريا» لا يطابق الواقع بفرق كبير + تنظيم أدوار المخازن + نقل توقيت الخصم لحالة «في التجهيز» + تحقّق مزامنة حالة سوريا + ربط مدير إعلانات بالحملات.

## القرارات (عبر AskUserQuestion)
- المخزن الثالث في سوريا = **مخزن واحد مشترك** للجملة/المناديب/المتاجر/الأمانة.
- تركيا = **مستودع كبير + مبيعات** (نفس نمط سوريا).
- تصحيح الجرد الحالي = **جرد فعلي يدوي** عبر شاشة الجرد.
- الترتيب = حسب رأي المنفّذ (Phase A أولاً).

## السبب الجذري لفرق الجرد
`reserveForOrder` تُخصم **لحظة إنشاء الطلب** بغضّ النظر عن الحالة → كل طلب «وارد جديد»/«بالانتظار» غير المُسلَّم يبقى مخصوماً للأبد → الجرد بالتطبيق أقل من الواقع.

## Phase A — بنية المخازن + توقيت الخصم

### A1. البنية الجديدة (per-market)
- سوريا: `central/syria` (مستودع كبير) → `sales/syria` (يوسف) + `wholesale/syria` (جملة وتوزيع مشترك).
- تركيا: `central/turkey` (مستودع كبير) → `sales/turkey` (فاطمة).
- SQL idempotent: إضافة نوع `wholesale`؛ تحويل المركزي الحالي → `central/syria`؛ إنشاء `central/turkey` + `wholesale/syria`. البضاعة تبقى بمكانها، الجرد الفعلي يضبط الأرقام.

### A2. توقيت الخصم — مُصلِّح حالة موحّد `syncOrderStock(order, performedBy)`
يُستدعى عند كل إنشاء/تغيير حالة، يضمن تطابق مخزون الطلب مع حالته (idempotent، net-aware = Σحجز − Σإرجاع):
- **مخصوم** (stock out): `preparing, ready, motor_prep, at_center, shipped, on_way, special_delivery, motor, prepaid, delivered, settled`.
- **في المخزن** (stock in / يُسترد): `pending, waiting, not_received, returning, returned, cancelled`.
- يدعم تجهيز↔انتظار↔تجهيز بلا ازدواج. تركيا+سوريا نفس المنطق.
- نقاط الاستدعاء: إنشاء طلب (handleSave) · تغيير حالة فردي (handleStatusChange) · ترحيل جماعي (advanceBatch) · حذف/إلغاء.
- المصدر: `_resolveSourceWarehouse` كما هو (profiles.warehouse_id ثم مخزن مبيعات السوق).

### A3. UX شاشة المخازن
- ترتيب الأعمدة هرمياً ضمن كل سوق (مركزي→مبيعات→جملة) + فلتر سوق (سوريا/تركيا/الكل).
- تسمية نوع `wholesale` (🏪) + تسمية المركزي «🏛️ مستودع».

## Phase B — تحقّق مزامنة حالة سوريا (تشخيص + إصلاح)
مسار الحالة بالاتجاهين: جدول→تطبيق (`onSheetEdit`→`sheet-to-app`، تغطية STATUS_MAP + توقيت المُشغّل) · تطبيق→جدول (`sync-order-to-sheet`). تأكيد الانعكاس بالطرفين وإصلاح سبب التأخير.

## Phase C — ربط مدير الإعلانات بالحملات (رؤية فقط)
عمود `manager_name` على `campaigns` + منسدلة بالنموذج (مديا باير/مدير) + عرض/تجميع تحليلي. ليس تشغيل حملات.

## التحقق
بعد كل Phase: `npm run build` + اختبار حيّ. النشر مباشرة (SQL/git) حسب صلاحيات المالك.
