# تصميم: ملخّص محاسبي للتسليمات (تبسيط وإكمال تاب «تسليمات الشهر»)

> التاريخ: 2026-06-07 · المالك: Ayoub (Hosam) · الحالة: مسوّدة للمراجعة
> (نسخة مبسّطة — أُلغيت فكرة سجل التدقيق/SLA السابقة بطلب المالك: «بدون تعقيدات».)

## 1. الهدف
تسهيل الشغل وجعل **إحصاء الطلبات واضحاً وسلساً**، وتمكين **المحاسب** من رؤية:
- إجمالي المبيعات المسلّمة
- المنتجات المباعة **وقيمها** (مش بس الكمية)
- عمولة كل موظف
كل ذلك بصفحة بسيطة قابلة للتصدير — **عبر تبسيط وإكمال تاب «📦 تسليمات الشهر» الموجود، لا بناء نظام جديد**.

## 2. الموجود حالياً (لا نكرّره)
تاب «تسليمات الشهر» ([OrdersScreen.jsx](src/screens/OrdersScreen.jsx)) يعرض فعلاً:
- تصفية الطلبات المسلّمة (status=delivered) للشهر الحالي
- تجميع حسب الموظف: عدد الطلبات + إجمالي بكل عملة + USD + المنتجات **(بالكمية فقط)**
- إجماليات عامة (USD + كل عملة)
- كروت العمولة لكل موظف (`EmployeeCommissionCard`)
- فلتر براند (LOWE'S/STRONG) + إعدادات العمولة

## 3. الناقص (نطاق هذا العمل)

### 3.1 قيمة المنتجات (Product values)
- **القيد:** بنود الطلب `{name, qty}` بلا سعر للوحدة؛ الطلب له `amount` إجمالي واحد.
- **الحل (إعادة استخدام):** جدول `product_economics` (موجود، يملؤه المالك: `item_name`, `sale_price_usd`)
  و`profitabilityService` يحسب أصلاً «الإيراد = السعر × الكمية».
- **المخرج:** قسم جديد «🧾 المنتجات المباعة» في التاب — جدول: المنتج · الكمية المباعة · **القيمة (USD)**
  = `units × sale_price_usd`. والمنتجات بلا اقتصاديات تظهر القيمة «—» مع تلميح «أضف سعرها».
- وفي كرت كل موظف: بجانب كل منتج نعرض قيمته أيضاً (مش بس ×الكمية).
- **توضيح مهم:** القيمة الإجمالية للمبيعات تبقى من `amount` الفعلي (الأدقّ)؛ قيمة المنتج تقديرية من
  أسعار الكتالوج (USD) — نوضّحها بنص صغير حتى ما تلتبس.

### 3.2 اختيار الفترة (Period selector)
- حالياً الشهر الحالي فقط. نضيف اختيار: **شهر سابق** أو **مدى تاريخي (من–إلى)**.
- نعمّم الفلتر `monthStart` ليصير `dateFrom/dateTo` (نمط موجود بشاشة الطلبات نفسها).
- العنوان والإجماليات تتبع الفترة المختارة.

### 3.3 تصدير/طباعة للمحاسب (Export)
- زر «⬇️ تصدير Excel» (مكتبة `xlsx` موجودة بالمشروع — `vendor-xlsx`) يصدّر ملفاً فيه 3 جداول:
  1. ملخّص: إجمالي المبيعات المسلّمة (لكل عملة + USD) + عدد الطلبات + الفترة
  2. المنتجات: المنتج · الكمية · القيمة (USD)
  3. الموظفون: الاسم · عدد التسليمات · المبيعات · العمولة
- (طباعة عبر المتصفح كخيار ثانوي لاحقاً — التصدير أهم للمحاسب.)

## 4. التقسيم والتسليم
كل خطوة مستقلة وقابلة للنشر وحدها:
1. **قيمة المنتجات** — قسم «المنتجات المباعة» + قيمة بجانب منتجات الموظف (إعادة استخدام `product_economics`).
2. **اختيار الفترة** — شهر/مدى مخصص بدل الشهر الحالي.
3. **تصدير Excel** — الجداول الثلاثة.

## 5. الوصول والأمان
- التاب يبقى للمدراء/الأدمن (كما هو). نضيف وصول **المحاسب** له (دور المحاسبة) إن لم يكن متاحاً.
- لا تغيير على نموذج/تدفق إدخال الطلب → مخاطرة منخفضة جداً.
- لا جداول جديدة. نعتمد `product_economics` الموجود.

## 6. الاختبار
- بعد كل خطوة: `npm run build` + اختبار حيّ (preview + حقن جلسة مدير).
- التحقق: مطابقة إجمالي المبيعات وقيم المنتجات والعمولات مع استعلام SQL مباشر لفترة معروفة.
- التأكد أن منتجاً بلا `product_economics` يظهر قيمته «—» دون كسر العرض.
