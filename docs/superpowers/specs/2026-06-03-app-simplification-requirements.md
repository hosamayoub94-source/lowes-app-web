# مبادرة تبسيط وتنظيم التطبيق — متطلبات (للجلسة القادمة)
> Requirements capture · 2026-06-03 · طلب المالك صراحةً (نشتغل عليها «مابعد»)

> ملاحظة: هذه **متطلبات محفوظة**، وليست تصميماً نهائياً. التصميم يُنجز بجلسة brainstorming مخصّصة قبل التنفيذ (حتى لا نزيد الفوضى الحالية).

## المشكلة (بكلمات المالك)
- كثرت **التابات والبنرات** في التطبيق → الترتيب «معجقة» ومشتّت.
- العناصر متفرّقة (مثال: العملاء المتكررون بمكان، الأرشيف بمكان).
- الموظفون يتشتتون بأدوات لا تخصّهم (مثال: المبيعات يرون خصائص السوشال).

## الرؤية
- **تبسيط للموظفين** · **تنظيم للإداريين** · وصول سهل لكل مستخدم لأدواته فقط.
- ثيم/ألوان مريحة احترافية — انطباع «منظومة بـ50,000$ مثل الشركات الكبيرة».
- التركيز على **سهولة الاستخدام للجميع**.

## المبادئ المطلوبة
1. **قوائم حسب الدور (role-based navigation):** كل دور يرى فقط أدواته (المبيعات لا ترى السوشال، إلخ). تقليل عدد عناصر التنقل لكل مستخدم.
2. **تقليل البنرات/التابات:** دمج المتشابه، إخفاء غير الضروري، تجميع منطقي.
3. **أتمتة + قوالب جاهزة قابلة للتحرير:** مثل القوائم المنسدلة (مدن/شحن/دفع — أُنجزت)، قوالب للمهام/الرسائل/الطلبات مع إمكانية التعديل دائماً.
4. **إلهام/اقتراح حسب سلوك المستخدم (personalization):** اقتراح الأدوات/الإجراءات حسب استخدامه.
5. **هوية بصرية احترافية:** لوحة ألوان متّسقة (navy/teal/cream الحالية أساس جيد)، تباعد مريح، اتساق المكوّنات.

## نطاق محتمل (يُحسم بالتصميم)
- إعادة هيكلة `navigation.js` لقوائم مختصرة per-role + تجميع (مجموعات قابلة للطي).
- مراجعة البنرات (PushPermissionPrompt، MockModeBanner، تنبيهات…) ودمج/تقليل.
- توحيد «العملاء/الأرشيف» في تجربة واحدة متماسكة.
- نظام قوالب عام (مهام/رسائل/طلبات).
- مراجعة الثيم/التباعد.

## مرتبط
- أُنجز جزئياً: قوائم منسدلة (مدن/شحن/دفع)، عملاء per-seller. 
- يُكمَّل: دمج المناديب (مرحلة 3 المخازن)، الثيم، القوائم per-role.
